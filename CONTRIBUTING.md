# Contributing to MATH2421 Probability Materials

Thank you for considering contributing to this repository! This project is an **unofficial backup** of course materials for MATH2421 (Probability) at HKUST. Your contributions can help enhance the learning experience for others.

## How You Can Contribute

### 1. Add New Materials

- Lecture notes
- Assignments and solutions
- Tutorial problems and solutions
- Past exam papers
- Additional resources (e.g., cheat sheets, summaries, etc.)

### 2. Improve Existing Content

- Fix typos or formatting issues in lecture notes or other materials.
- Update outdated information.
- Add missing solutions or explanations.

### 3. Share Your Insights

- Share your personal notes or ideas related to the course.
- Provide alternative solutions to problems.
- Suggest additional resources or references.

## Contribution Guidelines

### General Rules

- Ensure that your contributions are relevant to the course (MATH2421 Probability).
- Do not upload copyrighted materials unless you have explicit permission.
- Avoid sharing solutions to ongoing assignments or exams.

### File Naming

- Use clear and descriptive file names (e.g., `lecture1_notes.pdf`, `assignment1_solution.docx`).
- Follow the existing folder structure when adding new files.

### Formatting

- Use Markdown (`.md`) for text-based content whenever possible.
- Ensure that your content is well-organized and easy to read.

### Submitting Changes

1. Fork this repository.
2. Create a new branch for your changes:

   ```bash
   git checkout -b your-branch-name
   ```

3. Commit your changes with a clear message:

   ```bash
   git commit -m "Add lecture 5 notes"
   ```

4. Push your changes to your forked repository:

   ```bash
   git push origin your-branch-name
   ```

5. Open a pull request to the `main` branch of this repository.

Recommend to use VScode with the extension "GitHub Pull Requests" for an easier way to manage pull requests.

## Code of Conduct

By contributing, you agree to uphold our [Code of Conduct](CODE_OF_CONDUCT.md) to ensure a welcoming and respectful environment for all contributors.
